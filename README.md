 # 评价分权系统

> 纯桌面端评语归档工具 — 单文件 exe，双击即用，零配置。

---

## 一、项目概述

评价分权系统是一个 Windows 桌面应用，用于日常工作中的评价/评语录入与归档。核心功能：

- **评语录入**：支持文本评语 + 多张图片附件
- **智能归档**：按当天日期自动建主文件夹，按订单编码建子文件夹
- **批量模式**：一次粘贴多个订单号，逐个填写，支持跳过/返回
- **历史浏览**：内置表格浏览所有已归档记录，双击打开所在文件夹

### 为什么做这个

日常工作中需要对大量订单/工单写评语，文字+截图散落在聊天记录里，回头很难找。这个工具让你录入即归档，按日期→订单号两层目录自动整理，评语存为 `review.txt`，图片直接放进对应文件夹。

### 技术选型

| 层面 | 选择 | 理由 |
|------|------|------|
| GUI 框架 | tkinter (纯 Python) | 零外部 GUI 依赖，Windows 原生兼容 |
| 图片处理 | Pillow (PIL) | 剪贴板粘贴、缩略图、格式转换 |
| 打包 | PyInstaller (onefile) | 单 exe 分发，无需安装 Python |

没有 Flask、没有浏览器、没有 WebView2 —— 这是一个纯原生桌面窗口。

## 二、使用指南

### 2.1 安装 & 启动

1. 将 `评价分权系统.exe` 放到桌面（或任意位置）
2. 双击启动

首次启动会在 exe 同级目录自动创建 `评价分权系统数据/` 文件夹，所有数据都存在里面。

### 2.2 单订单录入

1. 默认在"录入"标签页，模式为**单订单**
2. 填写**订单编码**、**评语**内容
3. 点击**选择图片**或 **Ctrl+V 粘贴**添加截图
4. 点击**提交**

提交后文件归档到 `{数据目录}/reviews/2026-06-16/ORDER-001/`：
```
ORDER-001/
├── review.txt          ← 评语文本
├── img_1718000000_0.png  ← 截图
└── img_1718000000_1.jpg
```

### 2.3 多订单批量录入

1. 点击**多订单批量**切换到批量模式
2. 在文本框中粘贴订单号（一行一个，支持逗号/分号/换行分隔）
3. 点击**开始逐个填写**
4. 逐一为每个订单填写评语和图片
5. 点击**保存并继续**进入下一个，或**跳过**当前订单
6. 全部完成后弹出汇总

### 2.4 查看历史

- 切换到**历史**标签页，表格展示所有已归档记录
- 双击任意一行 → 自动打开该订单的文件夹
- 点击**刷新**重新扫描文件系统

### 2.5 更改存储位置

- 顶栏右侧点击**更改目录**，选择新的存储路径
- 设置会持久化保存，下次启动自动沿用

### 2.6 图片操作

- **选择图片**：打开文件对话框多选
- **剪贴板粘贴**：截图后直接 Ctrl+V 粘贴（也支持点击按钮）
- 缩略图预览：选中后展示缩略图，可点击「x 移除」删除单张

## 三、项目结构

### 3.1 源码目录

```
review-system/
├── app.py              # 主程序（唯一源码文件）
├── app.spec            # PyInstaller 打包配置
├── README.md           # 本文件
│
├── output/             # 编译产物
│   └── 评价分权系统.exe
│
├── 评价分权系统数据/    # 运行时数据（与 exe 同级）
│   ├── config.json     # 用户配置（存储路径等）
│   ├── error.log       # 异常日志
│   └── reviews/        # 归档根目录
│       └── 2026-06-16/ # 日期主文件夹
│           └── ORDER-001/  # 订单子文件夹
│               ├── review.txt
│               └── img_*.jpg
```

### 3.2 数据文件说明

| 文件 | 作用 |
|------|------|
| `config.json` | 保存用户自定义的存储路径 |
| `error.log` | 未捕获异常自动记录，方便排查 |
| `reviews/日期/订单号/review.txt` | 评语文本，追加写入 |
| `reviews/日期/订单号/img_*.jpg` | 图片附件，带毫秒时间戳防重名 |

### 3.3 打包配置（app.spec）

关键设计决策：

- **`runtime_tmpdir=None`**：临时文件使用系统 Temp 目录，避免在桌面散落 `_MEI*` 文件夹
- **`console=False`**：无命令行黑框，纯 GUI 体验
- **排除项**：Flask、numpy、pandas 等无关库被显式排除，减小 exe 体积
- **tcl/tk 数据文件**：手动收集，确保 tkinter 运行时不缺失字体/主题文件

### 4.2 核心模块拆解

#### `WinButton`（自定义按钮组件）

tkinter 原生 Button 在 Windows 上的视觉效果有限。`WinButton` 继承 `tk.Canvas`，用 `create_rectangle` + `create_text` 自绘：

- **三种状态**：normal / hover / pressed，各有独立配色
- **立体感**：pressed 状态下文字偏移 1px 模拟内凹效果
- **高光线**：hover 时顶部画白色细线增强立体感
- **accent 模式**：主题蓝实心按钮（提交/主操作）

关键实现：
```
_colors() → 返回 (文字色, 填充色, 边框色) 三元组
_redraw() → delete("all") 后重新绘制矩形+文字
configure(text=...) → 拦截自定义 text 参数，避免 Canvas 报错
```

### 4.3 关键设计决策

#### 1. 为什么用纯 tkinter 而不是 Web 前端

| 方案 | 问题 |
|------|------|
| Flask + 浏览器 | 需要启动本地服务器，弹浏览器窗口 |
| Flask + pywebview | 依赖 Edge WebView2，部分机器不兼容导致 exe 秒退 |
| Electron | 体积大（100MB+），过度设计 |

最终选择 **纯 tkinter**：单文件 20MB，任何 Windows 10/11 可直接运行。

#### 2. 数据统一收进子文件夹

所有运行时产物（config、log、reviews）都在 `评价分权系统数据/` 下，桌面只留 exe + 一个数据文件夹。避免早期版本中 `_MEI*` 临时目录和 `reviews/` 散落桌面的问题。

#### 3. 路径安全处理

`sanitize()` 函数过滤文件名非法字符（`<>:"/\|?*`），防止订单号中含特殊字符导致建目录失败。

#### 4. 图片格式兼容

保存时自动处理 RGBA→RGB 转换（JPEG 不支持透明通道），保留原始 PNG 的透明信息。

#### 5. 全局异常捕获

`sys.excepthook` 兜底，未捕获异常写入 `error.log` 并弹窗提示，避免静默崩溃。

### 4.4 依赖

| 依赖 | 用途 |
|------|------|
| `tkinter` | GUI 框架（Python 标准库） |
| `Pillow` | 图片加载、缩略图、剪贴板读取 |
| `PyInstaller` | 仅构建时使用，不进入运行时 |

## 五、构建 & 开发

### 环境要求

- Python 3.10+
- Pillow（`pip install Pillow`）
- PyInstaller（`pip install pyinstaller`）

### 调试运行

```bash
python app.py
```

```bash
pyinstaller --clean app.spec
```
